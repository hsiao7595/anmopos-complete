# AGENTS.md

## 專案名稱
御手足悦 × 足不老｜雙店營運數據管理系統

## 專案定位
這是一套給兩家按摩店使用的內部 POS／營運統計後台。
第一階段不是完整 POS 結帳系統，而是先完成「每日進客統計、班別統計、師傅統計、月報表、年報表」。

## 適用店家
1. 御手足悦按摩養生會館
2. 足不老養生莊園

## 第一階段核心目標
讓兩家店每天可以登入後台輸入每日營運數據，系統自動統計：
- 每日進客數
- 上半月統計
- 下半月統計
- 全月統計
- 年度統計
- 單店統計
- 兩店合併統計
- 兩店比較

## 技術架構指定
請使用：
- Next.js
- React
- TypeScript
- Tailwind CSS
- SQLite
- Prisma
- NextAuth 或自製登入機制
- xlsx 匯出 Excel
- PDFKit 或 Puppeteer 匯出 PDF

## 開發優先順序
請先完成 MVP，不要先做完整 POS。

優先順序：
1. 建立專案架構
2. 建立 Prisma schema
3. 建立資料庫 seed
4. 建立登入系統
5. 建立店家切換
6. 建立每日營運資料輸入
7. 建立月報表
8. 建立年報表
9. 建立 Excel 匯出
10. 建立 PDF 匯出
11. 建立列印樣式

## 禁止優先開發的功能
第一階段請不要先做以下功能：
- 完整 POS 結帳
- 會員儲值
- 折扣券
- 師傅抽成
- 現金／刷卡結帳
- 庫存管理
- 複雜圖表分析

這些是第二階段功能，請先保留擴充性即可。

## UI 風格
後台畫面需：
- 乾淨
- 清楚
- 像 Excel 報表
- 適合店長、櫃台、行政快速操作
- 不要太花俏
- 重點是好輸入、好查看、好列印

建議配色：
- 背景：淺灰／米白
- 表格：白底
- 統計區塊：淡藍、淡黃、淡粉
- 重要數字：深紫或深紅
- 按鈕：深色主按鈕

## 資料規則
- 同一家店同一天只能有一筆每日營運資料。
- 數字欄位不可輸入負數。
- 空值預設為 0。
- 修改資料時要更新 updated_at。
- 除數為 0 時，平均值與百分比請顯示 0 或 -，不可讓系統報錯。
- 匯出報表格式要接近 Excel 報表。
- 報表要適合 A4 列印。

## 開發完成定義
MVP 完成時必須符合：
1. 使用者可以登入後台。
2. 管理者可以看兩家店。
3. 店長只能看自己店。
4. 可以選擇御手足悦或足不老。
5. 可以新增每日營運資料。
6. 可以修改每日營運資料。
7. 系統會自動計算每日進客數。
8. 可以查詢指定月份報表。
9. 月報表會自動顯示上半月統計。
10. 月報表會自動顯示下半月統計。
11. 月報表會自動顯示全月統計。
12. 可以查詢年度報表。
13. 可以匯出 Excel。
14. 可以匯出 PDF。
15. 可以列印報表。

## 編碼慣例
- 使用 TypeScript。
- 優先使用清楚、容易維護的程式碼。
- API route 命名需直覺。
- 表單欄位需有基本驗證。
- 重要計算邏輯請集中成 utility function，避免散落在頁面中。
- 請建立 README，說明安裝、啟動、資料庫初始化與測試方式。
