# 前台 POS 系統啟動指南

## 第一次設置（筆電上）

```bash
cd frontend/anmopos
npm install
npm run dev
```

## 訪問系統

- **御手足悦**：http://localhost:5174/?store=yushou
- **足不老**：http://localhost:5174/?store=zubulao

## 編譯生產版本

```bash
npm run build
```

## 功能清單

### ✅ 現場師傅管理
- 師傅卡片顯示狀態和預約標記
- 點擊卡片切換師傅狀態（閒置→休息→服務中→下班）
- 優先級排序，空閒師傅優先顯示

### ✅ 房間位置管理
- **按摩房間**：A01~C02 (御手足悦), 01~13 (足不老)
- **按腳區**：14 個獨立位置 (F01~F14，只有御手足悦)
- **房間狀態**：
  - 🟢 空房 → 點「開始服務」
  - 🟡 已到客 → 點「切單」開始計時
  - 🔴 服務中 → 倒計時 + 點「結束」
  - 🟠 清潔中 → 點「完成清潔」
  - 🟣 預留 → 點「已到客」轉為服務

### ✅ 電話預約系統
- 「新增預約」按鈕建立預約
- 預約出現在「本日預約」tab
- 師傅卡片上顯示藍色預約 badge
- 「直接開單」快速轉為服務

### ✅ 切單功能
- 開單時勾選「待切單」
- 房間狀態顯示「已到客」（灰藍色）
- 點「切單」開始計時
- 師傅自動變更為「服務中」

### ✅ 倒計時
- 每秒更新流暢顯示
- 時間到時，房間卡片提示「時間已到，需要續單或結帳」
- 點「結束」後自動進入清潔狀態

## 快速操作流程

### 方案 A：快速開單（客人到場）
1. 房間選「開始服務」
2. 選擇師傅和服務項目
3. ✓ 開始計時

### 方案 B：待切單（客人未到）
1. 房間選「開始服務」
2. 勾選「待切單（客人已到但尚未開始）」
3. 客人泡腳/換衣完成→點「切單」
4. ✓ 開始計時

### 方案 C：電話預約（提前預約）
1. 點「新增預約」，填入客戶信息
2. 可選指定師傅和房間
3. 在「本日預約」tab 檢視
4. 客人到達→點「已到客」
5. 點「直接開單」快速開單
6. ✓ 開始計時

## 資料持久化

- 所有狀態保存在 localStorage
- 關閉瀏覽器後重新開啟會恢復最後狀態
- 模擬數據在 `src/data/` 目錄

## 多品牌支持

系統通過查詢參數 `?store=` 支持不同品牌：
- `yushou` - 御手足悦（推薦開始用的品牌）
- `zubulao` - 足不老

如需新增品牌，修改 `src/data/mockStores.js` 和 `src/data/mockRooms.js`

## 開發提示

- 修改模擬數據：`src/data/mock*.js`
- 新增品牌：`src/data/mockStores.js` + `src/data/mockRooms.js`
- 調整樣式：使用 Tailwind CSS（已配置）
- 狀態管理：Zustand store (`src/store/useAppStore.js`)
